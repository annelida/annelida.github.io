A virtual environment in Python is a separeted directory that contains a certain version of the Python interpreter with its own set of installed packages. The virtual environment is used to isolate the project from system-wide Python installation and from other projects. It is allows to avoid conflicts between packages.

To install virtual environment Python module venv used. Command 'python -m venv venv' will create a directory named 'venv'. 

python -m venv venv

To activate virtual environment, navigate to the created 'venv' and type: 

cd /home/vikky/venv/
source bin/activate

When the virtual environment is activated sys.path generates a list of paths from venv directory only. To see it let's create a simple script python_sys.py 

import sys
print(sys.path)

when activate a virtual environment and run the script. 

cd /home/vikky/venv/
source bin/activate
python python_sys.py

['/home/vikky/venv', '/usr/lib/python39.zip', '/usr/lib/python3.9', '/usr/lib/python3.9/lib-dynload', '/home/vikky/venv/lib/python3.9/site-packages']

To deactivate the current virtual environment use the command “deactivate”.

When virtual envoronment is deactivated sys.path will search paths outside venv directory. 

import sys
print(sys.path)

'', 'usr/lib/python39.zip', '/usr/lib/python3.9', '/usr/lib/python3.9/lib-dynload', '/home/vikky.local/lib/python3.9/site-packages', '/usr/local/lib/python3.9/dist-packages', '/usr/lib/python3/dist-packages']
